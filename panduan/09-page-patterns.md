# Page Patterns
Version: 1.0

# Purpose

Define consistent layouts for every major page type in the School Information System.

All new pages should follow one of these patterns.

---

# Pattern 1 - Dashboard

Structure:
1. Header
2. KPI Cards
3. Analytics
4. Operational Widgets
5. Activity Timeline

Reference:
06-dashboard.md

---

# Pattern 2 - List Page

Examples:
- Students
- Teachers
- Classes
- Inventory
- Subjects

Structure:
1. Page Header
2. Filter Card
3. Data Table
4. Pagination

Primary action:
Top-right "Add New"

---

# Pattern 3 - Detail Page

Examples:
- Student Detail
- Teacher Detail

Structure:
1. Breadcrumb
2. Header Card
3. Tabs
4. Detail Content
5. Activity Log

---

# Pattern 4 - Create/Edit Form

Structure:
1. Header
2. Form Sections
3. Sticky Action Bar

Support:
- Draft
- Validation
- Unsaved changes

Reference:
07-form-guidelines.md

---

# Pattern 5 - Wizard

Use for complex workflows.

Structure:
- Stepper
- Content
- Navigation Buttons

Maximum:
6 steps.

---

# Pattern 6 - Report Page

Structure:
1. Header
2. Filters
3. Charts
4. Tables
5. Export Actions

---

# Pattern 7 - Settings

Structure:
Left:
Navigation

Right:
Settings Panel

Use categories.

---

# Pattern 8 - Profile

Structure:
- Header
- Avatar
- Tabs
- Personal Information
- Preferences
- Security

---

# Pattern 9 - Notifications

List layout.

Newest first.

Support:
- Read
- Unread
- Filter

---

# Pattern 10 - Authentication

Pages:
- Login
- Forgot Password
- Reset Password

Minimal layout.

Single focus.

---

# Empty Pages

Every page must define:
- Empty
- Loading
- Error
- Permission denied

states.

---

# Common Header

Contains:
- Title
- Description
- Breadcrumb (optional)
- Primary Action

---

# Primary Actions

Place in top-right.

Never more than one primary button.

---

# Secondary Actions

Group into overflow menu when more than three.

---

# Cards

Use consistent spacing.

Radius:
28px

Gap:
24px

---

# Responsive

Desktop:
Multi-column

Tablet:
Reduced columns

Mobile:
Single column

---

# Success Criteria

Every page should:
- Be recognizable
- Use familiar patterns
- Minimize learning curve
- Support efficient daily workflows
